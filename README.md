# MiniLang Compiler

A **complete, multi-stage compiler** built in Python for a custom C-like language called **MiniLang**. Covers all 7 stages of compiler design — from lexical analysis to runtime environment — with an interactive web visualizer for presentation.

## Architecture

```
Source Code (.mini)
    │
    ▼
┌─────────────────── FRONT END ───────────────────┐
│  1. Lexer (lexer.py)          → Token Stream     │
│  2. Parser (parser.py)        → AST              │
│  3. Semantic (semantic.py)    → Annotated AST    │
└──────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────── MIDDLE END ──────────────────┐
│  4. IR Generator (ir_generator.py) → TAC         │
│  5. Optimizer (optimizer.py)       → Optimized TAC│
└──────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────── BACK END ────────────────────┐
│  6. Code Generator (codegen.py) → Bytecode       │
│  7. VM Runtime (vm.py)          → Output         │
└──────────────────────────────────────────────────┘
```

## Quick Start

### Run a program
```bash
python main.py run examples/fibonacci.mini
```

### See all compiler stages
```bash
python main.py all examples/factorial.mini
```

### Start the web visualizer
```bash
python server.py
# Open http://localhost:8080
```

## CLI Commands

| Command | Description |
|---------|-------------|
| `python main.py run <file>` | Compile and run |
| `python main.py tokens <file>` | Show lexer output |
| `python main.py ast <file>` | Show Abstract Syntax Tree |
| `python main.py semantic <file>` | Show symbol table |
| `python main.py ir <file>` | Show Three-Address Code |
| `python main.py optimize <file>` | Show optimized TAC |
| `python main.py bytecode <file>` | Show generated bytecode |
| `python main.py trace <file>` | Run with VM trace |
| `python main.py all <file>` | Show ALL stages |

## MiniLang Language

```c
func fibonacci(n: int) -> int {
    if (n <= 1) {
        return n;
    }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

func main() -> void {
    var i: int = 0;
    while (i < 10) {
        print(fibonacci(i));
        i = i + 1;
    }
}
```

### Features
- **Types**: `int`, `float`, `bool`, `string`, `void`
- **Variables**: `var name: type = value;`
- **Functions**: `func name(params) -> return_type { ... }`
- **Control Flow**: `if/else`, `while`, `for`
- **Operators**: `+ - * / %`, `== != < > <= >=`, `and or not`
- **Arrays**: `var arr: int[10];` with `arr[i]` access
- **I/O**: `print(expr)`, `input()`
- **Comments**: `// single line`, `/* multi-line */`

## Compiler Stages Explained

1. **Lexer** — Hand-written character scanner (no regex). Produces tokens.
2. **Parser** — Recursive descent (top-down, LL(1)). Produces AST.
3. **Semantic Analyzer** — Type checking, scope resolution, symbol tables.
4. **IR Generator** — Produces Three-Address Code (TAC).
5. **Optimizer** — Constant folding, propagation, DCE, CSE, strength reduction.
6. **Code Generator** — Translates TAC to stack-based bytecode.
7. **VM** — Stack-based virtual machine (runtime environment).

## No External Dependencies

Everything is hand-written from scratch in Python — no PLY, ANTLR, or any parser generators. This demonstrates true understanding of compiler construction.
